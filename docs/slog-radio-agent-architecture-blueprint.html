02 六类运行时节点 RUNTIME NODES
N01
SceneContextNode
归纳时间、天气、城市级位置和客户端状态；缺失上下文时只使用时间与用户控制项。
N02
ArchiveSliceNode
从声音档案挑选本次可用切片；Slog 侧不能读取完整档案。
N03
CandidateBuilderNode
聚合 MusicKit、mock、用户库和最近播放，只保留可播放候选曲。
N04
RadioPlannerNode
受限 ReAct：Observe、Plan、Act、Reflect；单次生成最多五个外部工具调用。
N05
RadioComposerNode
生成队列、短口播、解释与 Slog seed；TTS 失败时文本仍可播放。
N06
MemoryConsolidatorNode
把 skip、like、完成收听和转 Slog 事件整理成可审计的记忆候选。

04 工具调用矩阵 TOOL USE MATRIX
TOOL PERMISSION TIMEOUT FALLBACK
music.catalog.search MusicKit / network 1500ms 使用 `MockCatalog` 或 featured cache，保证电台仍可播放。

  • 客户端心跳包含当前曲目、队列位置、播放状态、网络和用户反馈。
  • 后端返回 `continue`、`replenish_queue`、`light_rerank`、`request_consent` 等轻量指令。
  • 30 分钟无心跳后结束会话，后台记忆整理不阻塞下一次播放。
06 记忆整理与隐私分层 MEMORY / PRIVACY
ephemeral
本轮请求有效，例如“今晚慢一点”。默认不落库。
session
当前电台会话有效，例如连续跳过高能歌曲。会话结束后汇总。
rolling_summary
7-30 天窗口偏好，可衰减、可覆盖，不用单次行为改长期偏好。
profile / sensitive
长期偏好需可见、可删；心理状态、精确位置、私人事件默认不保存。
